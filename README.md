# Край на пред история

Когато отвориш cms и видиш .venv- Веднага триене! Тя не ти трябва, винаги го правим от съображение за сиурност много ясно. Python Rage bait-ва много жестоко. ;) (Ако се чудиш на кой беше идеята (не споменавам имена) беше един от нашите учители)


1. Създай .venv с командата
бъди винаги в директорията на cms, когато изпълняваш
```bash
python -m venv .venv
```
2. Инсталиране на dependences
```bash
pip install -r requirements.txt
```
3. Правене на датабазата
Първо и най важо
имаш две папки в cms blog и cms
в блог е датабазата
в cms е логиката на сайте
за да направиш датабазата правиш следното:
отвори setting.py;
Намери променливата DATABASES
и направи това
```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',  # Указва, че ползваме MySQL
        'NAME': 'test_pgknma',  # името на датабзата в mysql
        'USER': 'root',  # Потребителско име root след това се защити от лоши хора *x
        'PASSWORD': '', # паролата първо за root после за защитения профила
        'HOST': 'localhost',  # IP на phpmyadmin/mysql
        'PORT': '3306',  # Порта като цяло винаги си е бил на 3306 ама ако не е смени

        # НЕ бараме
        'OPTIONS': {
            'init_command': "SET sql_mode='STRICT_TRANS_TABLES'",
            'charset': 'utf8mb4',
        }
    }
}
```
